# Transform Modes
Last updated: March 2025  ·  6 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/transform.mdx)

Loafer gives you three ways to transform your data. You can choose the mode that best fits your technical proficiency and the complexity of the transformation.

## Overview

| Mode | When to use | LLM required | Retry support |
|---|---|---|---|
| **AI** | Quick, messy cleanups, complex logical routing. | Yes | Automatic schema-checking |
| **SQL** | Established analytics queries, aggregations. | No | No (Syntax errors fail fast) |
| **Python** | Heavy custom logic, external API lookups. | No | Extensively testable |

## AI mode

Supply basic natural language instructions, and your configured LLM (Gemini, Claude, or OpenAI) generates raw Python code to handle the transformation locally.

### Supported Providers

Loafer is provider-agnostic. You can configure any of the following in your `pipeline.yaml`:

- **Google Gemini** (Default: `gemini-1.5-flash`)
- **OpenAI** (e.g., `gpt-4o`)
- **Anthropic Claude** (e.g., `claude-3-5-sonnet`)
- **Qwen** (via OpenAI-compatible API)

### How it works

1. **Schema Observation:** Loafer observes incoming records and extract column mappings.
2. **Privacy-First Prompting:** The schema and your instructions are sent to the LLM. **Raw row data is never sent to the LLM.**
3. **Local Execution:** The LLM returns a Python function which Loafer evaluates safely inside your local runtime.

> [!IMPORTANT]
> **Data Privacy:** Your production data stays 100% in your environment. The LLM only sees the "shape" of your data to write the logic.

### Writing good instructions

Good instructions yield performant code.

*   **Specify data types explicitly:** Instead of "fix the amounts", write "convert 'amount_kobo' to 'amount' divided by 100 as float."
*   **Handle nulls defensibly:** State clearly, "If 'address' is missing, set to 'Unknown'."
*   **Drop unwanted data boldly:** Tell it explicitly to prune out rows that don't match criteria.

### Understanding retries

If the generated AI function throws a runtime exception, the LLM catches the traceback error from Python and automatically issues a refinement correction attempt.

## SQL mode

Write transformations via standard ANSI SELECT queries. 

### The `{{source}}` placeholder

Because Loafer runs queries locally using sqlglot, the incoming stream is represented transparently under the `{{source}}` keyword matrix.

```yaml
transform:
  type: sql
  query: >
    SELECT
      id,
      amount / 100.0 AS amount_naira,
      UPPER(name)    AS name
    FROM {{source}}
    WHERE email IS NOT NULL
```

## Custom Python mode

If your transformation logic spans hundreds of lines or involves complex API integrations not fit for an LLM—write it yourself.

```yaml
transform:
  type: python
  file: ./scripts/complex_logic.py
```

Inside your `.py` file you must export a `transform(data: list[dict]) -> list[dict]` handler.
