# Target Connectors
Last updated: March 2025  ·  2 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/targets.mdx)

Loafer targets act as the endpoint of the pipeline, cleanly and idempotently saving the sanitized streaming data.

## PostgreSQL

A resilient PostgreSQL loader that uses high-throughput `COPY FROM` execution. 

### Configuration fields

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `url` | string | Yes | - | PostgreSQL URI. |
| `table` | string | Yes | - | Target table to load data. |
| `mode`| enum | No | append | Write operation mode `append`, `overwrite` or `merge`. |

### Complete example

```yaml
target:
  type: postgres
  url: postgresql://analytics:${DB_PASS}@localhost:5432/wh
  table: fact_orders
  mode: overwrite
```

> [!WARNING]
> Write mode `overwrite` evaluates table schematics and runs a hard `TRUNCATE` operations. Scheduled pipelines typically run `append`. Be very careful.

## MongoDB

A scalable NoSQL document target. Loafer handles connection pooling and automatic collection creation if it doesn't already exist.

### Configuration fields

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `url` | string | Yes | - | MongoDB Connection URI. |
| `database` | string | Yes | - | Target database name. |
| `collection`| string | Yes | - | Target collection name. |

### Complete example

```yaml
target:
  type: mongodb
  url: mongodb://${MONGO_USER}:${MONGO_PASS}@localhost:27017/
  database: loafer_db
  collection: customers_processed
```

## JSON

A local target system to flush records into standard `ndjson` (Newline Delimited JSON) or pure JSON array formats.

```yaml
target:
  type: json
  path: /out/reports.json
```
