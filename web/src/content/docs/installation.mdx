# Installation
Last updated: March 2025  ·  3 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/installation.mdx)

Loafer is distributed as a Python CLI package. It relies on a local environment for extraction and transformation.

## Requirements

- Python 3.11 or higher
- [uv](https://docs.astral.sh/uv/) (Recommended)

### 1. Install via uv (Recommended)

The best way to install Loafer is using `uv tool`. This isolates the CLI application and its dependencies, ensuring it doesn't conflict with other Python projects on your machine.

```bash
uv tool install loafer-etl
```

### 2. Install via pip

If you prefer standard Python tools, you can install Loafer using `pip`. 

```bash
pip install loafer-etl
```

> Note: We recommend using a global virtual environment to avoid cluttering your system Python environment.

## Verify installation

Once installed, verify the CLI is accessible and working:

```bash
loafer --version
```

```text title="Output"
Loafer v0.1.0
```

## Environment setup

Loafer uses environment variables to safely handle sensitive credentials like your database passwords and API keys.

Create a `.env` file in your project root:

```bash title=".env"
# Database Connection
DATABASE_URL=postgresql://user:password@localhost:5432/db

# LLM Providers (Choose yours)
GEMINI_API_KEY=AIzaSy...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
```

> [!TIP]
> **Supported Models:** Loafer officially supports Google Gemini, OpenAI, Anthropic (Claude), and Qwen. You only need to provide the key for the provider you configure in your `pipeline.yaml`.

> [!WARNING]
> **Security:** Never commit your `.env` file to version control. Add it to your `.gitignore` immediately.

## Docker

If you prefer not to install Python, or want to deploy Loafer in a containerized environment, use our official Docker image.

### 1. Pull the image
First, fetch the latest official build from the GitHub Container Registry:

```bash
docker pull ghcr.io/lupppig/loafer:latest
```

### 2. Run a pipeline
Execute your pipeline by mounting your local directory into the container. This ensures Loafer can read your YAML config and write output files back to your host machine:

```bash
docker run --rm -it \
  --env-file .env \
  -v $(pwd):/workspace \
  -w /workspace \
  ghcr.io/lupppig/loafer:latest \
  run pipeline.yaml
```

*   `--rm`: Automatically removes the container after the execution finishes.
*   `-v $(pwd):/workspace`: Maps your current directory to `/workspace` inside the container.
*   `--env-file .env`: Passes your credentials securely from your local file.

For more advanced container usage and `docker-compose` orchestration, see the [Docker reference](/docs/docker).
