# Source Connectors
Last updated: March 2025  ·  4 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/sources.mdx)

Loafer ships with deeply optimized, streaming-first connectors for the most common data systems. All extraction relies on cursor-based streaming to ensure memory stability on huge tables.

## PostgreSQL

The `postgres` source uses `psycopg` native server-side cursors to seamlessly page hundreds of thousands of rows locally without triggering Out of Memory (OOM) exceptions.

### Configuration fields

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `url` | string | Yes | - | Standard PostgreSQL connection string. |
| `query` | string | Yes | - | The query to yield rows. |
| `chunk_size`| number | No | 5000 | Count of rows to yield in a single fetch block. |

### Complete example

```yaml
source:
  type: postgres
  url: postgresql://admin:${DB_PASS}@localhost:5432/production
  query: "SELECT id, amount, created_at FROM purchases WHERE amount > 100"
  chunk_size: 10000
```

### Common errors

**`psycopg2.errors.UndefinedTable`**
The relation (table) doesn't exist, or you supplied a wrong schema in the connection URL.

## MongoDB

The `mongodb` source connects natively to your cluster and maps BSON types seamlessly to pure JSON types (e.g., stringifying `ObjectId`).

### Configuration fields

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `url` | string | Yes | - | Standard MongoDB uri string. |
| `database` | string | Yes | - | Name of database |
| `collection`| string | Yes | - | Target collection |

### Complete example

```yaml
source:
  type: mongodb
  url: mongodb+srv://cluster.mongodb.net
  database: ecom
  collection: users
```

> [!TIP]
> **Notes:** MongoDB `ObjectId` arrays and deeply nested BSON dates are recursively unwrapped and transformed into ISO 8601 strings during the extraction phase globally, so you don't even have to script an AI prompt for it.

## CSV & Excel

When extracting from standard system files, Loafer defaults to a line-by-line generator which is infinitely scalable across gigabytes of data.

```yaml
source:
  type: csv
  path: /data/input.csv
  delimiter: ","
```
