# Quickstart
Last updated: March 2025  ·  5 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/quickstart.mdx)

In this guide, we'll go from zero to a successfully executed AI-powered ETL pipeline in less than 5 minutes.

## What you'll build

We will construct a pipeline that:
1. Extracts raw customer data from a local CSV file.
2. Evaluates the data against an AI transform function defined in plain English to clean out unformatted phone numbers and unstandardized currency.
3. Loads the clean data into a standardized JSON file.

## Create a sample CSV

First, let's create a dirty dataset. Create a file named `customers_raw.csv`:

```csv title="customers_raw.csv"
id,name,email,amount_kobo,phone
1,Alice Smith,alice@example.com,15000,08123456789
2,Bob Johnson,bob@company.co,null,+2349088776655
3,Charlie Brown,,9000,08099887766
4,Diana Ross,diana@mail.com,20000,
```

## Create your pipeline config

Now, define your pipeline. Create a file named `pipeline.yaml` in the same directory:

```yaml title="pipeline.yaml"
version: 1.0
mode: etl

source:
  path: ./customers_raw.csv

transform:
  instruction: >
    Convert `amount_kobo` dividing by 100 and rename it to `amount_naira`.
    Normalize all phone numbers to the E.164 format (+234 format for Nigeria).
    Drop any rows where the email is null or empty.

target:
  path: ./customers_clean.json
```

> **How does this work?** Loafer auto-detects that `source` is a CSV from the `.csv` extension, `target` is JSON from the `.json` extension, and `transform` is AI mode from the `instruction` field. No `type:` needed.

Make sure you've [installed Loafer](/docs/installation) and exported your `GEMINI_API_KEY`. Once that is ready, kick off the job:

```bash
loafer run pipeline.yaml
```

```text title="Output"
  ╭─ Complete ─────────────────────────────╮
  │  3 rows loaded  ·  2.8s total          │
  │  1 filtered by transform               │
  │  312 tokens used                       │
  ╰────────────────────────────────────────╯
```

```python
def transform(data: list[dict]) -> list[dict]:
    # Generated by Gemini 1.5 Flash
    result = []
    for row in data:
        if not row.get("email"):
            continue
            
        row["amount_naira"] = float(row.pop("amount_kobo", 0)) / 100.0
        
        phone = str(row.get("phone", ""))
        if phone.startswith("0"):
            row["phone"] = "+234" + phone[1:]
            
        result.append(row)
    return result
```

Approve the function, and let Loafer finish the execution.

## What just happened

1. **Extraction:** Loafer's CSV connector parsed the records using pure streaming primitives, emitting records over an async channel.
2. **Schema Inference:** The orchestrator evaluated the schema directly from the incoming steam, generating a structural representation.
3. **AI Generation:** The orchestrator dispatched your request, along with the inferred schema, to the LLM agent. The agent wrote a python function mapped directly to your instructions.
4. **Execution:** The records passed through the active transform node which evaluated the newly generated code defensively.
5. **Load:** The sanitized records were streamed synchronously into the `json` file destination.
