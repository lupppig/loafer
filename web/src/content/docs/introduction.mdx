# Introduction
Last updated: March 2025  ·  2 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/introduction.mdx)

Loafer is an open-source, AI-powered ETL and ELT pipeline CLI tool. It removes the boilerplate code usually required to extract data from a database, clean it, and load it into a new destination. You describe your transformation in plain language, and Loafer writes the code, validates it, and runs the pipeline efficiently.

It is built specifically for data practitioners who are tired of writing the same 40 lines of Python connection and chunking code just to move a table from point A to point B.

## Who it's for

- **Data Engineers:** Stop writing boilerplate connectors and loading code. Bring your SQL or Python. Loafer handles the rest.
- **Data Analysts:** Transform and load your data using clear natural language instructions. No Python required.
- **Data Scientists:** Quickly move messy datasets into clean, structured formats before dropping them into your model training pipelines.
- **Backend Engineers:** Move production database syncs out of application cron jobs and into robust, transparent ETL workflows.

## How it's different

- **No infrastructure:** Runs entirely on your local machine, a CI runner, or a Docker container. There is no control plane to manage, and your data never leaves your environment.
- **AI-native, but safe:** The AI mode generates transformation functions locally using the schema, not the raw data. The LLM never sees your production data.
- **Transparent:** You can always inspect the generated Python code, edit it, and use it manually in the future.

Loafer sits cleanly between lightweight scripts and heavy frameworks like Airbyte or dbt. It's built for workflows that are too complex for a one-liner, but too simple to warrant standing up a full orchestration cluster.

## Core concepts

- **Pipeline**: The configuration file (`pipeline.yaml`) that defines the entire lifecycle of your data movement.
- **Source**: The system where your data originates (e.g., PostgreSQL, a CSV file, REST API).
- **Transform**: The logic applied to filter, clean, or shape the data (AI, SQL, or Custom Python).
- **Target**: The destination where the transformed data is written.
- **PipelineState**: The persisted record of what was extracted and loaded, enabling incremental syncs on future runs.
