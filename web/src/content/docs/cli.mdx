# CLI Reference
Last updated: March 2025  ·  4 min read  ·  [Edit on GitHub ↗](https://github.com/lupppig/loafer/edit/main/web/src/content/docs/cli.mdx)

The core interaction layer with Loafer is its CLI. Designed for clarity, performance, and robust feedback.

You can always view built-in help commands by running `loafer --help` or `loafer [command] --help` in your terminal.

---

## `loafer run`

Executes a data pipeline. This is the primary command you will use to push data through your configurations. 

When you execute this command, Loafer builds the source and target clients, automatically generates and compiles your AI or local python transformations, and begins streaming records via the specified generator.

```bash
loafer run [OPTIONS] [CONFIG_FILE]
```

<br />

### Arguments

- `CONFIG_FILE` *(Optional)*
  The path to your pipeline schema file. Defaults to `./pipeline.yaml` in the current directory if omitted.

<br />

### Options

| Flag | Shorthand | Type | Default | Description |
|---|---|---|---|---|
| `--verbose` | `-v` | bool | false | Streams heavily detailed execution logs to stdout. Displays complete Python AST derivations from AI prompts and all raw data frames. |
| `--dry-run` | - | bool | false | Performs full compilation and evaluates the transformation pipeline against source records without committing any output to the target database or file. |
| `--auto-approve` | `-y` | bool | false | Bypasses the human-in-the-loop prompt for AI-generated code. Required when executing AI components in CI environments. |

---

## `loafer schedule`

Persists the execution configuration to a local SQLite persistent layer directly managed by `APScheduler`.

This command allows you to define background jobs without keeping standard interactive terminal sessions open. The daemon executes jobs via the `schedule` schema settings.

```bash
loafer schedule [OPTIONS] [CONFIG_FILE]
```

<br />

### Arguments

- `CONFIG_FILE` *(Optional)*
  The path to your pipeline schema file.

<br />

### Options

| Flag | Shorthand | Type | Default | Description |
|---|---|---|---|---|
| `--name` | `-n` | string | *Hash* | Custom readable name for the job (e.g. `nightly-sync`). If omitted, Loafer assigns a UUID. |
| `--force` | `-f` | bool | false | Overwrites any existing scheduled job with the exact same name. |

---

## `loafer jobs`

Lists all persistent jobs in the background daemon queue. This provides an immediate overview of your entire ETL schedule map.

```bash
loafer jobs [OPTIONS]
```

<br />

### Example Output

```bash
loafer jobs
```

```text title="Output"
  JOB               NEXT RUN        LAST RUN     STATUS
  nightly-sync      in 4h 23m       2 hours ago  ✓ Success
  stripe-daily      in 14m          yesterday    x Failed
```

<br />

### Options

| Flag | Shorthand | Type | Default | Description |
|---|---|---|---|---|
| `--json` | - | bool | false | Returns the queue map strictly in JSON payload format suitable for programmatic integrations. |
| `--clear` | - | bool | false | Purges all registered jobs aggressively. |

---

## `loafer validate`

Performs strict syntax validation mapping against the configuration definitions without actually connecting to the sources or evaluating pipelines. 

Use this to ensure all environment variables resolve seamlessly inside container CI deployments before shipping.

```bash
loafer validate [OPTIONS] [CONFIG_FILE]
```

<br />

### Options

| Flag | Shorthand | Type | Default | Description |
|---|---|---|---|---|
| `--strict` | - | bool | false | Errors out completely if any configuration flag or experimental feature is deprecated. |
